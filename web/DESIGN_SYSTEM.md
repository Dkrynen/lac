# LAC — Design System

The single source of truth for LAC's brand and UI. **Undergrowth**: near-black
canvas, one green accent, a vein-leaf mark whose veins are circuit traces.
Hardware-first, tactile, half-plant-half-PCB. Dark-first. Information-dense
but calm.

> **Why "LAC"?** Short for **Local AI Companion**. One command (`lac`), one
> job: find the best model your machine can actually run, then prove it with
> real benchmarks.

---

## 1. Brand

| | |
|---|---|
| **Name** | LAC — Local AI Companion (lowercase wordmark: `lac`) |
| **Tagline (primary)** | local AI, sorted. |
| **Tagline (alt)** | The control room for your local models. |
| **Personality** | Hardware-first, tactile, quietly technical. |
| **Voice** | Sharp, technical, calm. No hype, no emoji. Say it once, precisely. |
| **Audience** | Developers and power users running local LLMs. |

**Logo — the vein-leaf.** A single-contour leaf outline whose veins are
right-angle circuit traces instead of organic curves. Meaning: *your machine,
mapped* — a leaf hiding circuitry. Canonical vectors: `assets/leaf-mark.svg`
(color) and `assets/leaf-mark-mono.svg` (currentColor). Raster sizes live in
`assets/icons/` and are regenerated by `assets/generate_icons.py` (Pillow,
dev-only). Scales favicon → app icon → README header → installer.

**Wordmark:** lowercase mono `lac`. In the web sidebar the wordmark's
blinking cursor block doubles as the accent dot. In the TUI (no SVG) the mark
degrades to a green `❋` glyph + `lac` wordmark in truecolor `#4ADE80` —
constants in `backend/brand.py`.

## 2. Color

Dark is the default. Tokens are CSS variables (see `tokens.css`); the Python
mirror for the TUI is `backend/brand.py`.

### Neutral (dark — default)
| Token | Value | Use |
|---|---|---|
| `--bg` | `#08090A` | App canvas (near-black) |
| `--surface` | `#0F1210` | Cards, sidebar |
| `--surface-2` | `#141917` | Inputs, raised rows |
| `--surface-3` | `#1B211D` | Hover, active well |
| `--overlay` | `rgba(4,5,4,.60)` | Modal scrim |
| `--border` | `rgba(228,232,226,.08)` | Hairlines |
| `--border-strong` | `rgba(228,232,226,.14)` | Emphasis borders |
| `--text` | `#E4E8E2` | Primary text (off-white) |
| `--text-muted` | `#7C8981` | Secondary text |
| `--text-faint` | `#545C55` | Tertiary / placeholders |

### Accent — Green (the one accent)
| Token | Value | Use |
|---|---|---|
| `--accent` | `#4ADE80` | Primary actions, focus, live/measured numbers |
| `--accent-hover` | `#6AEB9C` | Hover state |
| `--accent-pressed` | `#34C06A` | Active/pressed |
| `--accent-soft` | `rgba(74,222,128,.13)` | Tints, selected row bg |
| `--accent-fg` | `#06170D` | Text on accent (dark, not white) |

**Accent discipline:** green appears only where the instrument speaks —
primary actions, focus rings, live/measured numbers, the deep-dive toggle and
its revealed math, the TUI prompt/cursor. Never for decoration, never for
large fills.

### Semantic
| Token | Value | Use |
|---|---|---|
| `--success` | `#2DD4BF` | "Fits GPU", installed, OK — teal, deliberately not the accent green |
| `--warning` | `#D9A84C` | "Offload", partial — warm amber |
| `--danger` | `#E5484D` | "Too large", errors, destructive |
| `--info` | `#6FA8D8` | Info toasts (desaturated blue) |

### Light theme
Provided under `[data-theme="light"]` in `tokens.css`. Same semantic
structure; soft green-tinted paper canvas `#F6F8F4`, surfaces white/near-white,
borders `rgba(20,40,24,.10)`, text `#12201A`. Accent deepens to `#1FA157` for
contrast on paper (`--accent-fg` stays dark). Light is secondary — dark ships
first.

- **Sans:** Geist Sans → fallback Inter, system-ui. (Dev-tool standard.)
- **Mono:** Geist Mono → fallback JetBrains Mono, ui-monospace. Used for **model names, metrics, VRAM, params, code, tokens** — anything machine-y.

## 4. Spacing, radius, elevation, motion

- **Spacing:** 4px base grid (Tailwind defaults: 1=4, 2=8, 3=12, 4=16, 6=24, 8=32).
- **Radius:** `--radius-sm` 6px · `--radius` 8px (default) · `--radius-lg` 12px · pill 9999px. Small radii = serious tool.
- **Elevation:**
  - `--shadow-sm` `0 1px 2px rgba(0,0,0,.30)` — cards
  - `--shadow-md` `0 4px 12px rgba(0,0,0,.35)` — popovers
  - `--shadow-lg` `0 12px 32px rgba(0,0,0,.45)` — modals
  - `--shadow-focus` `0 0 0 2px var(--bg), 0 0 0 4px var(--accent-soft)` — focus ring
- **Motion:** 150ms ease-out default; 200ms for transforms (Tailwind key `ease-lac`). Honor `prefers-reduced-motion`.
- **Focus:** always a visible green ring. Never outline:none without replacement.

## 5. Components (primitives)

shadcn/ui base, themed to these tokens (Tailwind color names use `verdant*` —
see `tailwind.config.ts`):

- **Button** — variants: `primary` (verdant, dark text), `secondary` (surface-2 + border), `outline` (transparent + border), `ghost` (transparent, hover surface-3), `danger` (danger bg). Sizes: `sm` / `md` / `lg` / `icon`.
- **Badge** — semantic (`success`/`warning`/`danger`/`info`) + `neutral` + `accent`. Dot + label. Used for compatibility, status, capabilities.
- **Card** — `--surface`, 1px `--border`, `--radius`, `--shadow-sm`, 16–20px padding.
- **Input** — `--surface-2`, 1px border, 8px radius, green focus ring, mono optional.
- **Progress** — track `--surface-3`, fill `--accent` (or semantic). For download % and VRAM-fit bars.
- **Tabs · Tooltip · Dialog · Sheet · Select · Switch · ScrollArea · Skeleton · Toast(sonner) · Table** — standard.

---

## 6. Signature patterns

- **Model card** — name (mono, semibold), one-line description, a row of capability badges (vision/tools/thinking/embed), a compatibility verdict badge (Fits / Offload / Too large), params + context (mono, faint), and a thin VRAM-fit progress bar. Primary action: Install / Run.
- **Compatibility verdict** maps directly to semantic colors (success/warning/danger) — it's the most-glanced element, so make it scannable.
- **Density:** tables/lists lead; big hero art is avoided. Mono numerics align. Hover rows tint `--surface-3`.
- **Empty/loading states** use Skeletons, never spinners-with-no-context.

## 7. Implementation

- **Stack:** Vite + React + TypeScript + Tailwind CSS + shadcn/ui, source in `web/`.
- **Tokens:** `web/tokens.css` defines `:root` (dark) + `[data-theme="light"]`; `web/src/index.css` mirrors them for the bundle. Tailwind config maps `colors.*` to `var(--*)`. shadcn components consume the same vars. The TUI consumes the Python mirror `backend/brand.py`.
- **Engine:** unchanged — Flask (`backend/api.py`) stays the REST API + Ollama layer. In **dev**, Vite proxies `/api` → Flask (`:5050`). In **prod**, `vite build` → `web/dist`, Flask serves `web/dist` as static and falls through to `index.html` for client routing.

---

## 8. Accessibility

- Target **AA**: text-on-bg pairs verified ≥ 4.5:1. Green `#4ADE80` on `--bg` ≈ 11.5:1; `--accent-fg #06170D` on green ≈ 10.7:1. In the light theme, green `#1FA157` on paper is ≈ 3.4:1 — reserved for UI components and large text (≥ 3:1), never body copy.
- Visible focus everywhere; keyboard nav for every interactive element.
- `prefers-reduced-motion` disables transitions.
- `prefers-color-scheme: light` can auto-switch, but dark is default.
