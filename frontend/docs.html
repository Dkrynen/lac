API Reference

Complete reference for all Apt REST API endpoints. All responses are JSON unless otherwise noted. The server runs at http://127.0.0.1:5050.

System

GET /api/scan

Detect system hardware — OS, CPU, cores, RAM, GPU(s), VRAM, Apple Silicon status.

Example response
{
  "os": "Windows 10",
  "cpu": "AMD64 Family 25 Model 33",
  "cores": 16,
  "ram_gb": 32.0,
  "gpus": [{"name": "NVIDIA GeForce RTX 4090", "vram_gb": 24.0, "backend": "cuda"}],
  "total_vram_gb": 24.0,
  "is_apple_silicon": false,
  "in_container": false
}
GET /api/recommend?vram=24&use_case=coding&top_k=10

Get model recommendations based on VRAM, use case, and count. Uses live hardware scan if vram omitted.

ParamTypeDescription
vramfloatVRAM in GB (optional, defaults to scanned value)
use_casestringOne of coding, general, reasoning, chat
top_kintMax recommendations (default 5)
Example response
{
  "vram_gb": 24.0,
  "ram_gb": 32.0,
  "recommendations": [{
    "name": "Qwen3 14B",
    "model_id": "qwen3:14b",
    "provider": "ollama",
    "params_b": 14.8,
    "quant": "Q4_K_M",
    "score": 92,
    "vram_gb": 9.5,
    "context": 32768,
    "run_mode": "gpu",
    "ollama_cmd": "ollama run qwen3:14b",
    "scores": {"quality": 9, "speed": 8, "fit": 10, "context": 9}
  }]
}
GET /api/models

List all models in the curated database (65+ models). Returns id, name, provider, params, architecture, context, VRAM estimates.

GET /api/system/version

Returns app name, version, GitHub URL, and download URL.

Example
{"version": "2.2.0", "app_name": "Apt", "github_url": "https://github.com/Dkrynen/model-hub", "download_url": "https://github.com/Dkrynen/model-hub/releases"}
GET /api/system/check-update?current=2.2.0

Check GitHub for newer releases. Returns update_available, latest_version, download_url, and preview release_notes.

GET /api/system/ollama-path

Returns the filesystem path to the Ollama binary, or null if not found.

Ollama

GET /api/ollama/status

Check if Ollama is running. Returns {"running": true, "version": "0.5.0"} or {"running": false}.

GET /api/ollama/models

List locally installed Ollama models. Each entry has name, size_gb, modified, digest_short.

GET /api/ollama/ps

Show models currently loaded in memory. Returns {"running": true, "models": [...]}.

POST /api/ollama/pull

Download a model from the Ollama library. Streams server-sent events (SSE) with progress. Request body: {"model": "llama3.2:3b"}.

SSE stream example
data: {"status": "pulling manifest"}
data: {"status": "downloading", "completed": 12345678, "total": 98765432}
data: {"status": "verifying sha256 digest"}
data: {"status": "writing manifest"}
data: {"status": "success"}
data: [DONE]
POST /api/ollama/delete

Delete a locally installed model. Request body: {"model": "llama3.2:3b"}. Returns {"success": true}.

GET /api/ollama/check-install

Simple check: is Ollama on the system PATH? Returns {"installed": true, "path": "..."} or a download URL with instructions.

GET /api/ollama/check-install-detailed

Like check-install but also returns the Ollama version string from ollama --version.

Library

GET /api/library/browse?q=&capability=&sort=pulls&compatible=

Browse the Ollama library with filtering and sorting. Returns paginated model cards with VRAM estimates and compatibility info.

ParamTypeDescription
qstringSearch query
capabilitystringFilter by tools, vision, thinking, embedding, audio
sortstringpulls, name, vram, params
compatiblestringFilter by hardware fit: gpu, cpu
GET /api/library/tags?name=qwen3

Fetch available tag variants for a library model (e.g., :7b, :14b, :32b). Returns {"name": "qwen3", "tags": ["0.5b", "1.8b", "7b", "14b", "32b", "235b"], "count": 6}.

GET /api/ollama/library

Alias for /api/library/browse without parameters. Returns the full cached library.

Sessions

GET /api/sessions?workspace=default

List all saved sessions for the given workspace. Omit workspace to use currently active workspace.

POST /api/sessions

Create a new empty session. Body: {"name": "", "model": "qwen3:14b", "workspace": "default"}. Returns {"id": "abc123def45678"}.

GET /api/sessions/<session_id>

Fetch a session with all messages. Returns session metadata plus messages array.

PUT /api/sessions/<session_id>

Save messages to a session (replaces all existing messages). Body: {"model": "...", "messages": [...], "name": "My Chat", "workspace": "default"}.

DELETE /api/sessions/<session_id>

Delete a session and all its messages.

Workspaces

GET /api/workspaces

List all workspaces. Returns [{"id": "default", "name": "Default Workspace", "description": ""}].

POST /api/workspaces

Create a workspace. Body: {"name": "My Project", "description": "..."}. The id is auto-generated from the name.

GET /api/workspaces/<id>

Get a single workspace by id.

DELETE /api/workspaces/<id>

Delete a workspace and all its sessions. Cannot delete the default workspace.

POST /api/workspaces/<id>/switch

Switch the active workspace. Returns {"success": true, "workspace": "<id>"}.

Config

GET /api/config

Get current configuration. Returns {"workspace": "default", "ollama_host": "http://localhost:11434", "theme": "dark", "default_model": ""}.

PUT /api/config

Update configuration. Send any subset of keys: {"ollama_host": "http://192.168.1.100:11434", "default_model": "qwen3:14b"}.

GET /api/config/downloads

Get download history. Returns array of objects with model, status, size_gb, timestamp.

Chat

POST /api/ollama/chat

Streaming chat with an Ollama model. Body: {"model": "qwen3:14b", "messages": [{"role": "user", "content": "Hello"}]}. Returns SSE stream. Ends with data: [DONE].

SSE stream example
data: {"model":"qwen3:14b","message":{"role":"assistant","content":"Hello"},"done":false}
data: {"model":"qwen3:14b","message":{"role":"assistant","content":"! How"},"done":false}
data: {"model":"qwen3:14b","message":{"role":"assistant","content":" can I help you today?"},"done":true,"total_duration":123456789}
GET /

Serves the SPA frontend (frontend/index.html).

All endpoints require Ollama to be running (except /api/scan, /api/recommend, /api/models, /api/config, /api/workspaces, /api/sessions, and system endpoints).
Base URL: http://127.0.0.1:5050. Set OLLAMA_HOST environment variable to change the Ollama server address.

User Guide

Everything you need to know about using Apt — from first launch to advanced workspace management.

Quick Start

Prerequisites

  1. Install Ollama — Download from ollama.com/download and run the installer.
  2. Pull your first model — In a terminal, run ollama pull qwen3:7b (or any model from the library).
  3. Start Apt — Run python server.py from the project directory and open http://127.0.0.1:5050.

Starting the server

python server.py
python3 server.py
python3 server.py

The server listens on http://127.0.0.1:5050. The web UI loads automatically.

Using the CLI

python cli.py chat              # Start interactive chat
python cli.py list               # List installed models
python cli.py pull llama3.2:3b   # Install a model
python cli.py help               # Full command list

Hardware Scan

The hardware scanner automatically detects your system's components to determine which models will run well.

What gets detected

  • OS — Windows, macOS, or Linux
  • CPU — Architecture and core count
  • RAM — Total system memory in GB
  • GPU(s) — Name, VRAM, compute backend (CUDA, ROCm, Vulkan, Metal)
  • Apple Silicon — Unified memory detection
  • Container — Whether running in a container environment

Scan from the web UI

Open the Dashboard to see a live overview. Go to Scan & Recommend and click Scan Hardware for a detailed report.

Scan from the CLI

python cli.py scan

Output includes OS, CPU, cores, RAM, GPU details, VRAM total, and an estimate of the largest model that fits.

VRAM override

In the web UI, you can set a manual VRAM value in the Scan & Recommend page. This is useful when your GPU isn't detected automatically, you want to test different hardware, or you're sharing VRAM with other processes.

Recommendations

Apt scores 65+ models across four dimensions to find the best fit for your hardware:

ScoreWeightWhat it measures
Quality40%Benchmark performance, parameter count, architecture quality
Speed25%Tokens per second based on your hardware and model size
Fit20%How well the model VRAM matches your GPU
Context15%Available context window (larger is better for long documents)

How to use

  1. Go to Scan & Recommend in the web UI.
  2. Optionally set a VRAM override and use case.
  3. Click Scan + Recommend.
  4. Browse the scored list. Click Install on any model to download it.

Run modes explained

ModeBadgeMeaning
GPUGPUModel fits entirely in GPU VRAM (fastest)
OffloadOffloadPartially fits in VRAM; some layers run on CPU

CLI recommendations

python cli.py recommend --use-case coding --top-k 10
python cli.py rec                   # Shorthand
python cli.py recommend --use-case reasoning

Model Management

Installing models

From the web UI:

  • Browse Models — Search the library, filter by capability or hardware fit. Click Install on any model card.
  • Scan & Recommend — Click Install next to any recommendation.
  • Quick Install — Type a model name (e.g. qwen3:14b) and click Install.

From the CLI:

python cli.py pull qwen3:14b          # Download a model
python cli.py pull qwen3:14b-q4_K_M  # Specific quant

Viewing installed models

The Installed page lists all local models with name, size, and last modified date. Use the search box to filter. The Currently Running section shows models loaded in GPU memory.

Running a model

Click Run next to any installed model to copy the ollama run <name> command to your clipboard.

Deleting a model

python cli.py delete qwen3:14b
python cli.py rm qwen3:14b           # Shorthand

Download history

The Downloads page shows active downloads and a complete history of every model you've pulled. The CLI also tracks this: python cli.py config downloads.

Chat

The built-in chat provides a streaming conversational interface to any installed Ollama model.

Web chat

  1. Navigate to the Chat page.
  2. Select a model from the dropdown.
  3. Type your message and press Ctrl+Enter or click Send.
  4. The response streams in real time. Click Stop to interrupt.

CLI chat

python cli.py chat                    # Use first installed model
python cli.py chat qwen3:14b          # Specify a model
python cli.py chat --system "You are a helpful coding assistant."

Workspaces

Workspaces let you organize your work into separate environments. Each workspace has its own set of sessions.

Web UI

  • The workspace dropdown is in the sidebar, below the logo.
  • Select a workspace from the dropdown to switch.
  • Click the gear icon to create, rename, or delete workspaces.
  • Only the active workspace's sessions are shown.

CLI

python cli.py ws list                    # List workspaces
python cli.py ws create "Research"       # Create a workspace
python cli.py ws switch "Research"       # Switch to it
python cli.py ws show                    # See current workspace
python cli.py ws delete "Research"       # Delete (with confirmation)

Configuration is persisted to ~/.model-hub/config.json.

KeyDefaultDescription

CLI

python cli.py config show                  # View all settings
python cli.py config set ollama_host http://192.168.1.5:11434
python cli.py config set default_model qwen3:14b

Environment variable

Set OLLAMA_HOST to override the Ollama server address without changing the config file:

set OLLAMA_HOST=http://192.168.1.5:11434
python server.py

Troubleshooting

Ollama not found

  1. Make sure Ollama is installed from ollama.com/download.
  2. Start Ollama: look for the Ollama icon in your system tray or run ollama serve.
  3. Verify: curl http://localhost:11434 should return Ollama is running.

Server won't start

Make sure port 5050 isn't in use: netstat -ano | findstr :5050. Change the port by editing server.py.

GPU not detected

  • NVIDIA: Install the latest drivers (includes nvidia-smi).
  • AMD: Install AMD Software for Vulkan support.
  • Apple Silicon: Detection is automatic via sysctl.
  • Use the VRAM override field to manually enter your GPU memory.

Download fails

Large model downloads can fail on unstable connections. Try again — Ollama resumes interrupted downloads automatically.

Slow chat responses

First response may be slow while the model loads into GPU memory. If responses are consistently slow, try a smaller model or a lower quantization.
